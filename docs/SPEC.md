# 天国の涙 (Tears of Heaven) 仕様書

軽量タワーディフェンス × ローグライト・ハイブリッド。

- 中央の「生命の種」を、四方から押し寄せるゾンビから守るのが基本目標。
- プレイヤーは **カーソル(マウス位置)** を移動させることで範囲ダメージを与え、加えて **タワー** を設置して防衛する。
- ランごとに装備が落ち、ハブで合成・装着して次のランへ持ち越せる。レリックは永続強化。

## 1. 画面構成・基本パラメータ

## 2. ゲーム状態 (Phase)

```
hub ──(出撃)──> run ──(全wave クリア / 種破壊)──> result ──> hub
```

- **hub**: 装備一覧 / 合成 / レリック購入 / 出撃ボタン
- **run**: 実プレイ画面。カーソルでダメージ、タワー設置、Wave 進行
- **result**: 報酬計算(勝利時は涙 x2、装備持ち帰り)

## 3. 操作

| 操作 | 効果 |
|---|---|
| マウス移動 | カーソル(青い円)の位置を移動。範囲内の敵に持続ダメージ |
| タワー選択 → クリック | タワー設置(中心20px 範囲には設置不可) |
| `⚡追加` | 次の wave を即座にスポーンキューに追加 |
| `⚡全` | 残り全 wave をスポーンキューに一括追加 |

## 5. 敵タイプ (ETYPES)

通常スポーン時、wave に応じて以下の表からタイプ抽選。ボス(`wn % 10 === 0` かつ wave 最終1体)は常に基本扱い+HPx5。

| タイプ | HP倍率 | 速度倍率 | 装甲(タワー被ダメ) | 報酬倍率 | 出現開始 wave | 出現率 |
|---|---|---|---|---|---|---|
| 基本 (basic) | 1.0 | 1.0 | 100% | 1.0 | 1 | 残り |
| 速攻 (swift) | 0.5 | 1.6 | 100% | 0.7 | 4 | 20% |
| 装甲 (armor) | 1.6 | 0.7 | **50%** | 1.5 | 7 | 18% |
| 治癒 (healer) | 0.8 | 0.9 | 100% | 1.3 | 12 | 17% |

### タイプ別挙動

- **装甲**: タワー由来のダメージ(直接攻撃 / splash / chain / barricade)を半減。カーソルダメージは通常通り(=カーソル戦法に弱い)
- **治癒**: 1.2 秒ごとに半径 65px の味方を最大HPの 6% 回復
- **速攻**: HP半減と引き換えに 1.6 倍速。フロントラインの隙を縫う

### 視覚マーカー

- 装甲: 銀色の外周線
- 治癒: 半径 65px の薄い桜色の範囲リング
- ボス: 黄色のリング
- 全タイプ共通: HPが減ると赤くフレード

### スポーン全体スケール

```js
wZ(w)  = 6 + w * 3           // 1 wave あたりの敵数
wHp(w) = floor((8 + w * 5) * (1 + (w-1)*0.25))   // 基本HP
wSpd(w)= 20 + min(w*0.6, 30) // 移動速度(上限あり)
```

非ボス敵には個体差(HP `0.8x ~ 1.3x`、平均 1.05x)が掛かる。報酬と表示サイズも個体差に連動。

## 6. カーソルダメージ

`cD = 1 + (rlv[1]||0)*0.5 + et.cur_dmg_f`(=フラット)
`cDp = 1 + (rlv[1]||0)*0.03 + et.cur_dmg_p/100`(=乗算)
基本範囲 `cR = 36 + rlv[0]*4 + et.cur_range`

実効ダメージ/秒(範囲内の敵に対して):
```
ecD = (cD + 0.35 * 涙増幅器の数) * cDp * バフ補正
DPS ≈ ecD * 25
```

涙増幅器は範囲も +12 ずつ加算する。

ボスドロップは `BOSS_STATS`(より強力寄りのリスト)から抽選される。

### レアリティ抽選

```js
rollRar(w, boss, luck) {
  if (boss) return r < (30 + luck*3)% ? Legendary : Epic;
  const l = min(w*0.4, 20) + luck*2;
  // Legend  : 1 + l*0.15 %
  // Epic    : ~4%
  // Rare    : ~13%
  // Uncommon: ~27%
  // Common  : 残り
}
```

### 合成 (synthResult)

2 つの装備を1つに合成。

- スタットキーの和集合が 5 以下なら成立
- 同キー: `max(max(a,b), (a+b)/2 * 1.5)` → 同値なら 1.5x、別値なら最大値
- 異キー: 単に加算(=スタット数が増える)
- レアリティは高い方を継承

同値同士を繰り返し合成すると倍率は 1x → 1.5x → 1.88x → 2.16x → 2.37x …

## 10. 戦闘ループ要約

1 フレーム (`dt`) の主な処理:
1. バフ残り時間減算 → 期限切れを除去
2. 分身(`s.clones`)を更新(等速直線運動・中央矩形でバウンド・タイマー)
3. スポーンキュー進行
4. 種の自動回復
5. カーソル位置に水滴粒子を出す
6. 涙増幅器のオーラ効果(範囲・威力)を実効値に合算
7. 「希望の灯」のアタックバフをタワーに 1.5x で塗布(`dmgMap`)
8. タワー再生(「再生の苔」)
9. **タワー攻撃**(クールダウン式)
   - 命中時に splash / chain / heal / gold / slow 各 proc
   - 装甲タイプは被ダメ 50%
10. **ゾンビ更新**(以下を1個ずつ)
    - 攻撃中の処理 / 移動
    - 種接触 → 種ダメージ → 削除
    - カーソル範囲内ダメージ + クリティカル + マーク召喚
    - 分身範囲ダメージ(60% 倍率)
    - 治癒型なら 1.2s ごとに味方を回復
    - 壁スロー判定
    - HP <= 0 で onKill
11. 天罰 (`_wipeTriggered`)処理 — 全敵に最大HP 20%
12. 連鎖処刑(最大 5/frame)
13. パーティクル/ドロップ/弾/ダメ数字 の寿命減算とトリム
14. 描画

## 11. ドロップ持ち帰りルール

- 装備拾得は `dropR` 確率で発生し、`s.runDrops` に積まれる
- **クリア時(`tgt` 到達)**: 涙が 2 倍、全 `runDrops` が `stash` に移動
- **死亡時**: 涙のみ獲得、`runDrops` のうち 1〜3 個が `grave` として保存され、**次ラン中に同 wave 到達で復活**

## 15. Wave 1〜100 期待総体力 / 必要カーソルダメージ

期待値の計算前提:
- `wHp(w) = floor((8 + 5w) × (1 + (w−1)×0.25))`
- 個体差ばらつき: 平均 1.05x(範囲 0.8〜1.3x)
- タイプ別期待HP倍率:
  - w 1〜3 : 1.000(基本100%)
  - w 4〜6 : 0.900(基本80%+速攻20%)
  - w 7〜11: 1.008(基本62%+速攻20%+装甲18%)
  - w ≥ 12 : 0.974(基本45%+速攻20%+装甲18%+治癒17%)
- 10波毎の末尾1体はボス(基本HP × 5、タイプ抽選を経由しない)
- 装甲によるダメ減衰は **タワー攻撃のみ** に適用される(カーソルは装甲貫通)
